📐 Methodology

Gravity = Indispensability × 0.40 + Breadth × 0.30 + Design × 0.30

Gravity measures structural influence — how deeply an engineer's decisions are woven into the codebase architecture. It is not a measure of productivity or commit count.

Key metrics:

  • Indispensability — How much of the codebase depends solely on this person's code (bus factor)
  • Breadth — How widely the engineer's code spans across the project
  • Design — Ownership of architectural files (types, core modules, interfaces)
  • Quality — Code review & quality contribution signals
  • Survival — How much of the engineer's code survives rewrites over time
Generated by EIS (Engineering Impact Score) — OSS Gravity Map Project
